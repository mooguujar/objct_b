一、文档体系（分层规划）
1. 顶层规划文档
产品构想和规划书（你已有）
产品定位、用户画像、核心功能
技术架构选型
数据库设计
开发计划（分阶段）
2. 详细设计文档（按模块拆分）
每个模块一份文档，包含：
功能需求
接口设计（请求/响应）
数据库表结构
页面布局与交互
边界情况处理
3. 开发任务清单（可执行）
按优先级排序
每个任务包含：目标、输入、输出、验收标准
依赖关系标注
二、分阶段执行策略
阶段1：基础架构（1-2周）
任务清单：1. 项目初始化   - UniApp 项目创建   - Nuxt.js 项目创建     - NestJS 后端项目创建   - 数据库连接配置   2. 基础组件库   - 按钮、输入框、卡片等通用组件   - 统一样式变量   3. 用户系统基础   - 注册登录接口   - JWT 认证中间件   - 前端登录状态管理
阶段2：核心功能（按页面拆分）
每个页面独立任务：
Page 1: 首页发现页- 前端：瀑布流组件、卡片组件、下拉刷新- 后端：内容列表接口、分页逻辑- 数据库：查询优化- 验收：数据正常展示、交互流畅
阶段3：功能完善
数据统计埋点
后台管理系统
性能优化
三、给 AI 的提示词结构
标准任务格式：
【任务编号】页面名称开发【上下文】- 参考文档：@产品构想和规划书.md 第X章节- 设计图：@设计图/页面1.png- 相关代码：@src/pages/home/index.vue【需求】1. 实现首页发现页瀑布流2. 支持下拉刷新和上拉加载3. 卡片点击跳转详情页【技术要求】- 使用 uView UI 组件库- 使用 Pinia 管理状态- 接口路径：/api/posts/list【验收标准】- 数据正常展示- 交互流畅无卡顿- 代码符合项目规范【输出】- 页面代码文件路径- 需要新增的接口说明- 测试要点
四、关键规划要点
1. 模块化拆分
按功能模块拆分（用户、内容、岛屿、统计）
按页面拆分（10个页面独立开发）
按技术层次拆分（前端、后端、数据库）
2. 依赖关系明确
依赖图示例：用户系统 → 内容系统（需要用户ID）内容系统 → 岛屿系统（帖子关联岛屿）所有系统 → 数据统计（埋点依赖）
3. 接口先行
先定义接口文档（Swagger/OpenAPI）
前后端并行开发
使用 Mock 数据先行
4. 数据模型设计
完整的数据库 ER 图
每个表的字段说明
关联关系清晰
五、执行建议
1. 每次只给 AI 一个明确任务
❌ 错误：帮我开发整个项目
✅ 正确：帮我开发首页发现页的瀑布流功能
2. 提供完整上下文
引用相关文档章节
提供设计图或参考代码
说明技术选型和约束
3. 迭代式开发
先实现核心功能
再完善细节和优化
最后处理边界情况
4. 及时验证
每个任务完成后测试
发现问题立即修复
记录问题和解决方案

七、实际执行示例
假设现在要开发"首页发现页"，给 AI 的提示词应该是：
请根据 @产品构想和规划书.md 中"首页-发现页"的说明，参考 @设计图/page1.png 的设计，使用 UniApp + uView UI 开发首页发现页。具体要求：1. 实现顶部导航栏（关注/发现/热门岛屿三个标签）2. 实现双列瀑布流布局3. 实现下拉刷新和上拉加载更多4. 卡片点击跳转到详情页技术约束：- 使用 Pinia 管理列表数据- 接口调用使用 @src/utils/request.js- 样式使用 @src/styles/variables.scss 中的变量请先给出实现方案，确认后再开始编码。
八、质量控制
代码规范：统一 ESLint/Prettier 配置
组件复用：提取公共组件，避免重复
错误处理：统一错误处理机制
日志记录：关键操作记录日志
测试数据：准备测试数据脚本
总结：让 AI 精准完成项目的关键是：
文档完整（产品规划、技术设计、接口文档）
任务拆分（小任务、明确目标）
上下文充分（引用文档、设计图、代码）
迭代验证（逐步开发、及时测试）
规范统一（代码规范、命名规范、架构规范）
