# 岛屿社交平台产品构想和规划书

## 一、产品定位与核心理念

### 品牌定位

「岛屿社交平台」是一个以兴趣社区为核心的社交内容平台，通过"岛屿"概念连接志趣相投的用户，让每个人都能找到属于自己的精神岛屿，分享生活、交流经验、发现同好。

### 目标用户

- 18-35 岁城市年轻群体（学生/白领/自由职业者/创业者）
- 有特定兴趣爱好或专业领域的用户
- 希望找到同好、分享经验的创作者

### 核心价值主张

"每个人都是一座岛屿，在这里找到你的同类。"

通过兴趣标签和社区机制，让用户轻松发现感兴趣的内容和人群，形成有温度的社交连接。

## 二、App 总体架构

| 一级导航                   | 功能定位                       |
| -------------------------- | ------------------------------ |
| 首页（发现/关注/热门岛屿） | 内容浏览、关注动态、岛屿发现   |
| 岛屿                       | 加入的岛屿、岛屿详情、内容浏览 |
| 通知                       | 系统通知、互动通知、付费通知   |
| 我的                       | 个人中心、金币系统、创作管理   |
| 发布                       | 发布个人动态、发布岛屿帖子     |

## 三、页面结构与功能说明

### 1️⃣ 首页 - 发现页（Page 1）

**核心目标：** 通过瀑布流形式展示多样化内容，让用户快速发现感兴趣的内容

**页面布局：**

- **顶部导航栏**

  - 三个标签：关注 / 发现 / 热门岛屿（可切换）
  - 激活标签：黑色加粗字体，底部绿色下划线装饰
  - 搜索入口（右上角图标）
  - 渐变背景：从 `#d2f9e4` 经 `#e2f7ff` 到 `#f4f7ff`
  - 粘性定位（sticky top-0）

- **主内容区 - 瀑布流**

  - 双列卡片布局（columns-2，间距 8px）
  - 卡片样式：白色背景、圆角 12px、阴影
  - 内容类型：
    - 影视推荐卡片（含豆瓣评分标签、图片数量标识、简介文字覆盖层）
    - 生活分享卡片（图文混排，多图展示）
    - 话题讨论卡片（问答形式，带背景色块）
  - 每个卡片包含：
    - 封面图片/视频（带渐变遮罩层 from-black/50）
    - 标题和描述（底部覆盖层，白色文字带阴影）
    - 发布者头像和昵称（20px 圆形头像）
    - 点赞数（灰色图标和数字）
    - 图片数量标识（黑色半透明背景，显示 "+N"）

- **底部导航栏**
  - 白色圆角卡片（rounded-full，宽度 92%）
  - 固定底部，高度 56px
  - 导航项：首页（激活状态青色） / 岛屿 / 通知 / 我的
  - 中央悬浮加号按钮（黑色圆形，64px，阴影，位于底部 20px）

**交互细节：**

- 下拉刷新内容
- 无限滚动加载
- 点击卡片进入详情
- 点击头像进入用户主页

### 2️⃣ 首页 - 关注页（Page 2）

**核心目标：** 展示已关注用户的动态，形成个性化内容流

**页面布局：**

- **顶部导航栏**

  - 关注标签（激活状态，22px 加粗字体，带绿色波浪下划线 SVG 装饰）
  - 发现 / 热门岛屿标签（18px 灰色字体，可切换）
  - 搜索入口（右上角图标）
  - 渐变背景：从 `#e8fccf` 经 `#d7f9f7` 到 `#d3f4ff`

- **关注动态流**

  - 单列卡片流（间距 16px）
  - 卡片样式：白色半透明背景（bg-white/80），毛玻璃效果（backdrop-blur-sm），圆角 16px
  - 每条动态包含：
    - 用户头像和昵称（40px 圆形头像，昵称加粗）
    - 发布时间（12px 灰色文字）
    - 更多操作按钮（右上角三点图标）
    - 动态内容（文字/图片/视频）
    - 互动按钮（点赞、评论、收藏、分享，灰色图标）
    - 互动数据（点赞数、评论数，14px 灰色文字）

- **内容类型**
  - 纯文字动态（多行文字，支持 emoji）
  - 图文动态（图片下方文字）
  - 问答形式（带"HELP"标识，浅色背景卡片，大号加粗标题，红色波浪下划线装饰关键词）

- **底部导航栏**
  - 白色半透明圆角卡片（rounded-full，毛玻璃效果）
  - 带缺口设计（中央预留空间给加号按钮）
  - 中央悬浮加号按钮（灰色渐变，64px，位于导航栏上方）

**交互细节：**

- 点击用户头像查看主页
- 点击动态进入详情页
- 实时更新关注用户的新动态

## 十一、部署方案

### 前端部署

#### APP 端（UniApp）

- **开发环境：** 本地开发服务器
- **测试环境：** 测试服务器
- **生产环境：**
  - 打包为各平台应用（iOS、Android、H5）
  - iOS：提交 App Store
  - Android：提交应用商店或直接分发
  - H5：部署到 CDN

#### Web 端（Nuxt.js）

- **开发环境：** 本地开发服务器（npm run dev）
- **测试环境：** 测试服务器（SSR 模式）
- **生产环境：**
  - SSR 模式：部署到 Node.js 服务器
  - SSG 模式：静态生成后部署到 CDN
  - 推荐：SSR 模式用于动态内容，SSG 用于静态页面

### 后端部署

- **开发环境：** 本地开发
- **测试环境：** 测试服务器
- **生产环境：** 云服务器（阿里云/腾讯云）

### 数据库部署

- **开发环境：** 本地 MySQL
- **测试环境：** 测试数据库服务器
- **生产环境：** 云数据库（RDS）

### 文件存储

- **OSS 配置：** 阿里云 OSS
- **CDN 配置：** 阿里云 CDN 加速
