# 岛屿社交平台产品构想和规划书

## 一、产品定位与核心理念

### 品牌定位

「岛屿社交平台」是一个以兴趣社区为核心的社交内容平台，通过"岛屿"概念连接志趣相投的用户，让每个人都能找到属于自己的精神岛屿，分享生活、交流经验、发现同好。

### 目标用户

- 18-35 岁城市年轻群体（学生/白领/自由职业者/创业者）
- 有特定兴趣爱好或专业领域的用户
- 希望找到同好、分享经验的创作者

### 核心价值主张

"每个人都是一座岛屿，在这里找到你的同类。"

通过兴趣标签和社区机制，让用户轻松发现感兴趣的内容和人群，形成有温度的社交连接。

## 二、App 总体架构

| 一级导航                   | 功能定位                       |
| -------------------------- | ------------------------------ |
| 首页（发现/关注/热门岛屿） | 内容浏览、关注动态、岛屿发现   |
| 岛屿                       | 加入的岛屿、岛屿详情、内容浏览 |
| 通知                       | 系统通知、互动通知、付费通知   |
| 我的                       | 个人中心、金币系统、创作管理   |
| 发布                       | 发布个人动态、发布岛屿帖子     |

## 三、页面结构与功能说明

### 1️⃣ 首页 - 发现页（Page 1）

**核心目标：** 通过瀑布流形式展示多样化内容，让用户快速发现感兴趣的内容

**页面布局：**

- **顶部导航栏**

  - 三个标签：关注 / 发现 / 热门岛屿（可切换）
  - 搜索入口
  - 渐变背景（绿色到蓝色）

- **主内容区 - 瀑布流**

  - 双列卡片布局
  - 内容类型：
    - 影视推荐卡片（含评分、图片数量、简介）
    - 生活分享卡片（图文混排）
    - 话题讨论卡片（问答形式）
  - 每个卡片包含：
    - 封面图片/视频
    - 标题和描述
    - 发布者头像和昵称
    - 点赞数
    - 图片数量标识

- **底部导航栏**
  - 首页 / 岛屿 / 通知 / 我的
  - 中央悬浮加号按钮（发布入口）

**交互细节：**

- 下拉刷新内容
- 无限滚动加载
- 点击卡片进入详情
- 点击头像进入用户主页

### 2️⃣ 首页 - 关注页（Page 2）

**核心目标：** 展示已关注用户的动态，形成个性化内容流

**页面布局：**

- **顶部导航栏**

  - 关注标签（激活状态，带下划线装饰）
  - 发现 / 热门岛屿标签（可切换）
  - 搜索入口

- **关注动态流**

  - 单列卡片流
  - 每条动态包含：
    - 用户头像和昵称
    - 发布时间
    - 动态内容（文字/图片/视频）
    - 互动按钮（点赞、评论、收藏、分享）
    - 互动数据（点赞数、评论数）

- **内容类型**
  - 纯文字动态
  - 图文动态
  - 问答形式（带"HELP"标识）

**交互细节：**

- 点击用户头像查看主页
- 点击动态进入详情页
- 实时更新关注用户的新动态

  - 页面停留时长统计
  - 页面访问来源统计
  - 页面访问设备统计（APP/Web）

- **点击事件统计**

  - 按钮点击记录（按钮 ID、点击位置、点击时间、用户 ID）

  - 链接点击记录（链接地址、点击时间、来源页面）
  - 交互行为统计（点赞、评论、分享等）

- **数据上报机制**

  - 实时上报：关键操作立即上报
  - 批量上报：非关键操作批量上报（减少请求频率）
  - 本地缓存：网络异常时本地缓存，恢复后上报
  - 数据去重：防止重复上报

- **统计分析**
  - 页面访问热力图
  - 用户行为路径分析
  - 功能使用率统计
  - 转化率分析

## 五、技术架构

### 前端技术栈

#### APP 端（UniApp）

- **框架：** UniApp
- **UI 组件库：** uView UI / ColorUI
- **状态管理：** Vuex / Pinia
- **网络请求：** axios
- **图片处理：** 图片压缩、裁剪
- **视频处理：** 视频压缩、预览
- **统计埋点：** 页面访问统计、点击事件统计

#### Web 端（Nuxt.js）

- **框架：** Nuxt.js 3
- **UI 组件库：** Element Plus / Ant Design Vue
- **状态管理：** Pinia
- **网络请求：** axios / $fetch
- **服务端渲染：** SSR / SSG
- **SEO 优化：** 服务端渲染支持
- **统计埋点：** 页面访问统计、点击事件统计

### 后端技术栈

- **框架：** NestJS
- **ORM：** Prisma
- **数据库：** MySQL
- **文件存储：** 阿里云 OSS
- **图片处理：** Sharp
- **认证：** JWT
- **缓存：** Redis（可选）

### 数据库设计

#### 核心数据表

1. **用户表（User）**

   - id, username, nickname, avatar, phone, email
   - password_hash, status, role, created_at, updated_at

2. **岛屿表（Island）**

   - id, name, description, cover, avatar, category
   - owner_id, price, member_count, post_count
   - status, created_at, updated_at

3. **帖子表（Post）**

   - id, user_id, island_id, title, content
   - media_type, media_urls, like_count, comment_count
   - status, created_at, updated_at

4. **评论表（Comment）**

   - id, post_id, user_id, content, parent_id
   - like_count, created_at, updated_at

5. **关注表（Follow）**

   - id, follower_id, following_id, created_at

6. **加入岛屿表（IslandMember）**

   - id, user_id, island_id, join_type, paid_amount
   - created_at, updated_at

7. **通知表（Notification）**

   - id, user_id, type, title, content, related_id
   - is_read, created_at

8. **金币交易表（CoinTransaction）**

   - id, user_id, type, amount, balance, description
   - created_at

9. **页面访问统计表（PageView）**

   - id, user_id, page_path, page_title, device_type, platform
   - referrer, stay_duration, created_at

10. **点击事件统计表（ClickEvent）**

    - id, user_id, event_type, element_id, element_type, page_path
    - click_position_x, click_position_y, related_id, created_at

11. **用户行为统计表（UserBehavior）**
    - id, user_id, behavior_type, target_type, target_id, page_path
    - device_info, ip_address, created_at

- 常用间距：8px, 12px, 16px, 24px, 32px
- 页面边距：16px
- 卡片内边距：12px-16px

### 圆角系统

- 小圆角：8px（按钮、输入框）
- 中圆角：12px（卡片）
- 大圆角：16px-24px（弹窗、大卡片）

### 阴影系统

- 轻阴影：0 2px 8px rgba(0,0,0,0.04)
- 中阴影：0 4px 12px rgba(0,0,0,0.08)
- 重阴影：0 8px 24px rgba(0,0,0,0.12)

## 七、交互设计规范

### 动效系统

| 场景     | 动效表现    | 时长  | 曲线     |

| 按钮点击 | 透明度变化  | 150ms | linear   |
| 卡片出现 | 淡入 + 缩放 | 200ms | ease-out |
| 下拉刷新 | 旋转动画    | 持续  | linear   |
| 加载状态 | 骨架屏      | -     | -        |

### 反馈机制

  - 按钮点击：视觉反馈（透明度/颜色变化）

  - 错误操作：错误提示

- **加载反馈**
  - 页面加载：骨架屏

  - 下拉刷新：刷新动画

## 十一、部署方案

### 前端部署

#### APP 端（UniApp）

- **开发环境：** 本地开发服务器
- **测试环境：** 测试服务器
- **生产环境：**
  - 打包为各平台应用（iOS、Android、H5）
  - iOS：提交 App Store
  - Android：提交应用商店或直接分发
  - H5：部署到 CDN

#### Web 端（Nuxt.js）

- **开发环境：** 本地开发服务器（npm run dev）
- **测试环境：** 测试服务器（SSR 模式）
- **生产环境：**
  - SSR 模式：部署到 Node.js 服务器
  - SSG 模式：静态生成后部署到 CDN
  - 推荐：SSR 模式用于动态内容，SSG 用于静态页面

### 后端部署

- **开发环境：** 本地开发
- **测试环境：** 测试服务器
- **生产环境：** 云服务器（阿里云/腾讯云）

### 数据库部署

- **开发环境：** 本地 MySQL
- **测试环境：** 测试数据库服务器
- **生产环境：** 云数据库（RDS）

### 文件存储

- **OSS 配置：** 阿里云 OSS
- **CDN 配置：** 阿里云 CDN 加速
